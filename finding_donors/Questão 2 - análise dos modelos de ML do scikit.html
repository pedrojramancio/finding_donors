1 Support Vector Machines (SVM)

    1. Descreva uma situação do mundo real onde este modelo pode ser utilizado.

      1. Verificar e-mails e rotulá-los como urgentes ou não. Classificar se uma empresa é solvente ou não. Análise de ações da bolsa de valores.

    2. Quais são as vantagens da utilização deste modelo; quando ele performa bem?

      1. Oferece ótima acurácia nos dados de treinamento

      2. Mais eficiente na classificação de dados futuros

      3. Não faz suposições forte sobre os dados.

      4. Eles também são bastante robustos contra overfitting, especialmente em espaços de alta dimensão.

    3. Quais são as fraquezas do modelo; quando ele performa mal?

      1. No entanto, os SVMs consomem mais recursos computacionais;

      2. São mais difíceis de ajustar devido à importância de escolher o kernel correto

      3. Não escalam bem em conjuntos de dados maiores.

    4. O que torna este modelo um bom candidato para o problema, considerando o que você sabe sobre o conjunto de dados?

      1. Como queremos classificar os dados de uma forma binária (>50 mil ou não) podemos usar o SVM para tal, seria como se comportasse como uma regressão logistica) e não temos um grande voluma de dados. Contudo, a vantagem aqui é que pode se ter uma distribuição de pontos maior que 2D.

    5. Referência

      1. SVMs - An overview of Support Vector Machines - https://www.svm-tutorial.com/2017/02/svms-overview-support-vector-machines/

      2. Support Vector Machines (SVM) as a Technique for Solvency Analysis - https://core.ac.uk/download/pdf/6302770.pdf



2 Logistic Regression

    1. Descreva uma situação do mundo real onde este modelo pode ser utilizado.

      1. Usado para aplicar uma classificação binária. Indicar se um e-mail é spam ou não. Se uma paciente tem uma determinada doença ou não.

    2. Quais são as vantagens da utilização deste modelo; quando ele performa bem?

      1. É muito eficiente quando há um região que pode ser bem divida por uma reta;

      2. Não requer muitos recursos computacionais;

      3. Saída bem calibradas com as probabilidades preditas.

    3. Quais são as fraquezas do modelo; quando ele performa mal?

      1. Não conseguir resolver problemas não lineares, desde que sua superfície de decisão é linear. Ou seja, se não há uma área clara que possa ser divida por uma reta na distribuição dos pontos, o algoritmo não deve ser aplicado.

      2. Não é um dos mais poderosos algoritmos e pode ser superado por algoritmos mais complexos.

      3. Alta dependência da representação adequada de seus dados. Isso significa que a regressão logística não é uma ferramenta útil, a menos que você já tenha identificado todas as variáveis independentes importantes.

      4. É também um algoritmo conhecido por sua vulnerabilidade ao overfitting.

    4. O que torna este modelo um bom candidato para o problema, considerando o que você sabe sobre o conjunto de dados?

      1. Usado para aplicar uma classificação discreta. Então como é esse o objetivo de nosso problema este método se aplica bem, pois queremos dizer se alguém ganhar acima de 50 mil (1) ou não (0).

    5. Referências

      1. The Logistic Regression Algorithm - https://machinelearning-blog.com/2018/04/23/logistic-regression-101/

      2. Modern Machine Learning Algorithms: Strengths and Weaknesses - https://elitedatascience.com/machine-learning-algorithms

3 K-Nearest Neighbors (KNeighbors)

    1. Descreva uma situação do mundo real onde este modelo pode ser utilizado.

      1. Como funciona através da classificação de um elemento pela proximidade de suas características em relação a dos outros elementos, poderia citar como elemento a classificação de espécies de pássaros. Onde posicionar seu empreendimento, data a distância dos clientes.

    2. Quais são as vantagens da utilização deste modelo; quando ele performa bem?

      1. O algoritmo é simples e fácil de implementar.

      2. Não há necessidade de construir um modelo, ajustar vários parâmetros ou fazer suposições adicionais

      3. O algoritmo é versátil. Pode ser usado para classificação ou regressão

      4. Alta acurácia para modelos simples.

    3. Quais são as fraquezas do modelo; quando ele performa mal?

      1. O algoritmo fica significativamente mais lento à medida que o número de exemplos e / ou preditores / variáveis independentes aumenta.

      2. Computacionalmente caro

      3. Sensível aos outliers

    4. O que torna este modelo um bom candidato para o problema, considerando o que você sabe sobre o conjunto de dados?

      1. Pelo fato do modelo funcionar com um ajuste mínimo, ou seja não é preciso fazer suposições sobre os dados, e termos um volume pequeno, acredito que seja interessante usarmos este modelo.

    5. Referência

      1. A Practical Introduction to K-Nearest Neighbors Algorithm for Regression (with Python code) - https://www.analyticsvidhya.com/blog/2018/08/k-nearest-neighbor-introduction-regression-python/

      2. Machine Learning Basics with the K-Nearest Neighbors Algorithm - https://towardsdatascience.com/machine-learning-basics-with-the-k-nearest-neighbors-algorithm-6a6e71d01761

      3. A Quick Introduction to K-Nearest Neighbors Algorithm - https://medium.com/@adi.bronshtein/a-quick-introduction-to-k-nearest-neighbors-algorithm-62214cea29c7



4 Gaussian Naive Bayes (GaussianNB)

    1. Descreva uma situação do mundo real onde este modelo pode ser utilizado.

      1. Realizar uma análise quanto sabemos que uma dada informação tem uma determinada probabilidade de ocorrer. Por exemplo, realizar a previsão do tempo; descobrir nas redes sociais quem provavelmente é torcedor do “Chicago Bulls” para oferecer produtos do time.

    2. Quais são as vantagens da utilização deste modelo; quando ele performa bem?

      1. É fácil e rápido para prever o conjunto de dados da classe de teste. Também tem um bom desempenho na previsão de classes múltiplas.

      2. Quando a suposição de independência prevalece, um classificador Naive Bayes tem melhor desempenho em comparação com outros modelos como regressão logística, e você precisa de menos dados de treinamento.

      3. O desempenho é bom em caso de variáveis categóricas de entrada comparada com a variáveis numéricas. Para variáveis numéricas, assume-se a distribuição normal (curva de sino, que é uma suposição forte).

      4. Usar quando:

        1. Se tiver um conjunto de dados grande ou moderado

        2. Se as instâncias tiverem vários atributos

        3. Caso haja independência entre os atributos do conjunto de dados.

    3. Quais são as fraquezas do modelo; quando ele performa mal?

      1. O modelo pode falhar de forma gritante, pois é usado para analisar dados não numéricos, por exemplo, ao procurar por “Chicago bulls”, pode encontrar muita coisa da cidade Chicago e búfalos, quando na verdade se quer o time de basquete.

      2. Se a variável categórica tem uma categoria (no conjunto de dados de teste) que não foi observada no conjunto de dados de treinamento, então o modelo irá atribuir uma probabilidade de 0 (zero) e não será capaz de fazer uma previsão. Isso é muitas vezes conhecido como “Zero Frequency”. Para resolver isso, podemos usar a técnica de alisamento. Uma das técnicas mais simples de alisamento é a chamada estimativa de Laplace.

      3. A suposição de preditores independentes, na vida real, é quase impossível que ter um conjunto de indicadores que sejam completamente independentes.

    4. O que torna este modelo um bom candidato para o problema, considerando o que você sabe sobre o conjunto de dados?

      1. Por não ser um conhecedor profundo dos dados, acredito que este modelo não seja o mais indicado para o problema.

5 Decision Trees (Árvores de decisão)

    1. Descreva uma situação do mundo real onde este modelo pode ser utilizado.

      1. Classificar a espécie de um novo espécime dadas suas características, por exemplo, classificar as flores de Íris, dados largura e comprimento da pétala e da sépala da flor.

    2. Quais são as vantagens da utilização deste modelo; quando ele performa bem?

      1. As árvores de decisão constituem uma técnica muito poderosa e amplamente utilizada em problemas de classificação. Uma das razões para que esta técnica seja bastante utilizada é o fato do conhecimento adquirido ser representado por meio de regras.

      2. A árvore de decisão é simples de entender e visualizar.

      3. Pode usar dados numéricos ou não.

      4. Se os dados de treinamento não tem certo valores, então a árvore de decisão pode ser usada, pois podem manipular dados ausentes analisando outras colunas.

    3. Quais são as fraquezas do modelo; quando ele performa mal?

      1. A construção de uma árvore de decisão, chamado de processo de indução, pode ser uma tarefa de alta demanda computacional.

      2. Há muitas maneiras de uma árvore de decisão ser estruturada a partir de um conjunto de atributos. De forma exaustiva, o número de árvores de decisão possíveis cresce fatorialmente à medida que o número de atributos aumenta. Logo, torna-se impraticável definir a estrutura da árvore de decisão ótima para um determinado problema, devido ao elevado custo computacional envolvido nessa busca.

      3. Não trabalham bem se os dados não são correlacionados.

    4. O que torna este modelo um bom candidato para o problema, considerando o que você sabe sobre o conjunto de dados?

      1. Uma vez que tenha identificado os labels mais relevantes pode-se rapidamente montar uma regra para classificar os novos doadores, mas como não conheço bem os dados, acho que esse modelo não é indicado.

    5. Referências

      1. Árvores de Decisão -
        https://medium.com/machine-learning-beyond-deep-learning/%C3%A1rvores-de-decis%C3%A3o-3f52f6420b69

      2. Árvores de Decisão - ftp://ftp.dca.fee.unicamp.br/pub/docs/vonzuben/ia004_1s10/notas_de_aula/topico7_IA004_1s10.pdf



6 Ensemble Methods (Bagging, AdaBoost, Random Forest, Gradient Boosting)

    1. Descreva uma situação do mundo real onde este modelo pode ser utilizado.

      1. Predição de empréstimo bancário

    2. Quais são as vantagens da utilização deste modelo; quando ele performa bem?

      1. Ensembling é um método comprovado para melhorar a precisão do modelo e funciona na maioria dos casos.

      2. Torna o modelo mais robusto e estável, garantindo desempenho decente nos casos de teste na maioria dos cenários.

      3. Pode-se usar para capturar relações complexas lineares e simples, além de não lineares e complexas, nos dados. Isso pode ser feito usando dois modelos diferentes e formando um conjunto de dois.

    3. Quais são as fraquezas do modelo; quando ele performa mal?

      1. Reduz a capacidade de interpretação do modelo e dificulta a elaboração de qualquer visão de negócios crucial no final.

      2. É demorado e, portanto, pode não ser a melhor ideia para aplicativos em tempo real.

      3. A seleção de modelos para criar um conjunto é uma arte que é realmente difícil de dominar.

    4. O que torna este modelo um bom candidato para o problema, considerando o que você sabe sobre o conjunto de dados?

      1. Por se aplicável é vários tipos de cenários diferentes, resolveria nosso problema. Contudo, é um modelo complexo para ser aplicado a nosso cenário.

    5. Referência

      1. Ensemble methods - https://scikit-learn.org/stable/modules/ensemble.html
